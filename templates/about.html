About This Catalog

Collections

The catalog supports any number of named libraries — independent physical locations such as a home bookshelf, an office collection, or a storage unit. One library can be marked as Primary, which causes it to open automatically on the home page. Libraries marked as Storage show a distinct amber indicator on their volumes and expose a Box Number field for tracking which box a volume is packed in.

Creating, renaming, and deleting libraries is restricted to the owner account.

Entries & Volumes

A catalog entry is a pure bibliographic record — it describes a work, not a physical object. The same entry can have copies in multiple libraries with no duplication of bibliographic data.

Each physical copy is tracked as a volume, which records the library it lives in, its {{ labels.about_page_call_number }}, barcode, volume number (for multi-volume works), acquisition date, and condition flags for oversize, missing, and fragile items. Volumes can also carry an optional physical description — dimensions, weight, page count, binding style, colour, material, and freeform notes — shown as a collapsible panel on the entry detail page.

Digital copies are tracked as digital resources, each with a URL (supporting https:// and zotero:// schemes), an optional call number, and a type of either Full Text or Table of Contents.

Contributors (authors, editors, translators, compilers, illustrators, and other contributors) are tracked as a structured list on each entry, with an explicit role per person. Multiple contributors can be recorded, ordered, and edited independently. The contributor list feeds directly into the Chicago citation generator.

Eleven entry types are supported:

  • Book — ISBN, ISSN, Series, Series Number, Edition, Original Year, Original Publisher, Publisher, Publisher Location
  • Book Section — Book Title, Pages, ISBN, Series, Publisher, Publisher Location
  • Journal — ISSN, Publisher
  • Journal Article — Publication, Volume, Issue, Pages, ISSN
  • Map
  • Pamphlet
  • DVD / CD / VHS Tape
  • Conference Proceedings — Conference Name, Date, and Location
  • Government Document

All entries also carry Title, Subtitle, Year, Language, Notes, Condition, Acquisition Date, Acquisition Source, Abstract, and a {{ labels.about_page_classification_code }} field for the full {{ labels.about_page_call_number }} assigned during cataloguing. Subject access points are linked controlled vocabulary headings rather than free-text tags — see Subject Headings below.

Patrons & Access Control

The catalog supports multiple named patron accounts. Login is by patron number and password. Five roles control what each patron can see and do:

  • Owner — full access to everything, including patron and library administration. Only one owner is permitted.
  • Cataloguer — can view all entries and make all cataloguing changes; cannot add, remove, or configure libraries or patrons.
  • Regular — can browse and search unrestricted and restricted entries; cannot make changes.
  • Restricted — same browse permissions as Regular.
  • Guest — can browse and search unrestricted entries only.

Unauthenticated visitors have guest-level access. The owner sets and resets patron passwords from the Manage page.

Each entry carries a visibility field — Unrestricted, Restricted, or Hidden — which controls which roles can see it in search results, library views, and on the entry detail page.

Subject Headings

Subject access is managed through a controlled vocabulary modelled on the {{ labels.classification_schema }} subject heading system. Each heading is a distinct record with an optional type (Region, People, Person, Topic, Event, Language, or Work), a scope note, and structured relations to other headings:

  • BT / NT — Broader Term / Narrower Term (automatically reciprocated)
  • RT — Related Term (automatically reciprocated)
  • USE / UF — preferred form / non-preferred form (automatically reciprocated)
  • SA — See Also (one-directional)
  • SE — See Entry (one-directional)

Entries may only be tagged with preferred forms. If a non-preferred (USE) heading is entered, the system automatically substitutes the preferred form and notifies the cataloguer with a flash message.

The subject heading browse page is accessible to Regular patrons and above. Guests and unauthenticated visitors do not have access to the subject index. Creating, editing, and deleting headings is restricted to Owner and Cataloguer accounts.

Each heading detail page lists all entries tagged with that heading, and related headings can be previewed in a side panel without leaving the page.

Circulation Desk

The Checkout Desk provides a full circulation workflow for owner and cataloguer accounts. Items are added to a bin by scanning their barcodes, then checked out to a patron by patron number, checked back in, or transferred to a different library. Transfers automatically close any open checkout record for the moved volume.

The On Loan page shows every currently checked-out item alongside the borrowing patron and checkout time. Each patron's full checkout history is available from their record.

Search

Full-text search spans every bibliographic and physical field: title, author, year, language, publisher, subject headings, series, ISBN, ISSN, {{ labels.about_page_call_number }}, {{ labels.about_page_classification_code_lower }}, barcode, volume number, and date. Results can be filtered by collection and entry type. Search results respect the access level of the logged-in patron — hidden or restricted entries are filtered automatically.

Advanced Search allows multi-field boolean queries using AND and NOT operators across individual fields or all fields at once.

The Barcode Scan mode accepts a single barcode (typed or scanned with a hardware reader) and jumps directly to the matching entry. The input is focused automatically and submits on Enter, so a barcode scanner works hands-free.

Citations

A Chicago-style citation is automatically generated for every entry and shown at the top of its detail page.

Physical Audit

Audit mode lets you verify which volumes are physically present in a given library. A barcode scanner (or keyboard) is used to scan each item; the tool provides immediate colour-coded feedback (found, already scanned, not found) and tracks progress with a live counter. At the end of a session, a printer-friendly report lists all volumes that were not scanned.

Import & Export

Bulk import from a LibraryThing JSON export (librarything.json placed in the application directory). Single-volume entries are created automatically with a sequential barcode. Multi-volume books are split into numbered volumes; other multi-volume entries are flagged for manual review.

The catalog can be exported at any time as a ZIP archive containing four CSV files: the raw libraries, entries, and volumes tables, plus a fully joined catalog sheet with one row per volume.

Backups

Every write operation automatically saves a backup of the database. Short-term backups (the last 5 changes) are kept in backups/short/. Daily backups (the last 30 days) are kept in backups/daily/.
